# Contributing to Cutline OS

First off, thank you for considering contributing to Cutline OS! It's people like you that make Cutline OS such a great tool for creative agencies and professionals.

## Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## Fork & create a branch

If this is something you think you can fix, then fork Cutline OS and create a branch with a descriptive name.

## Get the test suite running

Make sure your changes don't break anything by running linting and build steps:
```bash
npm run lint
npm run build
```
See the `README.md` for local development setup, including environment variables and database initialization.

## Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help; everyone is a beginner at first.

## Code Style

- We use Next.js App Router and Server Components.
- We strictly type everything using TypeScript.
- We use Tailwind CSS v4 and shadcn/ui for styling.
- Please follow the existing patterns and refer to the Architecture & API Documentation in the Wiki.

## Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with Cutline OS's master branch. Then push your branch and open a Pull Request.

Please fill out the Pull Request template completely so we understand your changes.
